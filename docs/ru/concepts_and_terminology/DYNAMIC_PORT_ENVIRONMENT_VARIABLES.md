# Переменные окружения для динамических портов

Каждый экземпляр Coast получает набор переменных окружения, которые предоставляют [динамический порт](PORTS.md), назначенный каждому сервису. Эти переменные доступны как внутри обычных сервисов, так и внутри compose-контейнеров, и позволяют вашему приложению во время выполнения определить свой внешний доступный порт.

## Соглашение об именовании

Coast формирует имя переменной из логического имени сервиса в вашем разделе `[ports]`:

1. Преобразовать в верхний регистр
2. Заменить неалфавитно-цифровые символы на подчёркивания
3. Добавить `_DYNAMIC_PORT`

```text
[ports] key          Environment variable
─────────────        ────────────────────────────
web             →    WEB_DYNAMIC_PORT
postgres        →    POSTGRES_DYNAMIC_PORT
backend-test    →    BACKEND_TEST_DYNAMIC_PORT
svc.v2          →    SVC_V2_DYNAMIC_PORT
```

Если имя сервиса начинается с цифры, Coast добавляет к имени переменной префикс в виде подчёркивания (например, `9svc` становится `_9SVC_DYNAMIC_PORT`). Пустое имя заменяется на `SERVICE_DYNAMIC_PORT`.

## Пример

Дан следующий Coastfile:

```toml
[ports]
web = 3000
api = 8080
postgres = 5432
```

Каждый экземпляр Coast, созданный из этой сборки, будет иметь три дополнительные переменные окружения:

```text
WEB_DYNAMIC_PORT=62217
API_DYNAMIC_PORT=55681
POSTGRES_DYNAMIC_PORT=56905
```

Фактические номера портов назначаются во время выполнения `coast run` и различаются для каждого экземпляра.

## Когда их использовать

Наиболее распространённый сценарий использования — настройка сервисов, которые встраивают собственный URL в ответы: callback-адреса аутентификации, URI перенаправления OAuth, источники CORS или URL вебхуков. Этим сервисам нужно знать порт, который используют внешние клиенты, а не внутренний порт, на котором они слушают.

Например, приложению Next.js, использующему NextAuth, требуется, чтобы `AUTH_URL` был установлен на внешний доступный адрес. Внутри Coast Next.js всегда слушает на порту 3000, но порт на стороне хоста является динамическим:

```toml
[services.web]
command = "cd /workspace && AUTH_URL=http://localhost:${WEB_DYNAMIC_PORT:-3000} yarn dev:web"
port = 3000
```

Резервное значение `:-3000` означает, что команда также работает и вне Coast, где `WEB_DYNAMIC_PORT` не задана.

## Приоритет

Если переменная окружения с таким же именем уже существует в контейнере Coast (задана через secrets, inject или compose environment), Coast не перезаписывает её. Существующее значение имеет приоритет.

## Доступность

Переменные динамических портов внедряются в окружение контейнера Coast при запуске. Они доступны для:

- Команд `install` обычных сервисов
- Процессов `command` обычных сервисов
- Контейнеров compose-сервисов (через окружение контейнера)
- Команд, запускаемых через `coast exec`

Значения не изменяются в течение всего времени жизни экземпляра. Если вы остановите и снова запустите экземпляр, он сохранит те же динамические порты.
